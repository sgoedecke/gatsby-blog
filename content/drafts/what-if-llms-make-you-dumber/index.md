I don't think there's compelling evidence that using AI makes you less intelligent[^1]. However, it seems pretty obvious that using AI to perform a task means you don't learn as much _about performing that task_. That's the point, right? Instead of taking the time and effort to do it by hand, you can get a language model to do it for you.

Some software engineers think this is a decisive argument against the use of AI. The argument goes something like this:

1. Using AI means you don't learn as much from your work
2. AI-users thus become less effective engineers over time, as their technical skills atrophy
3. Therefore we shouldn't use AI in our work

This doesn't follow. It would really suck for software engineers if using AI made us worse at our jobs in the long term. But we might still be obliged to use it, if it provided enough short-term benefits.

Until around 2024, the best way to learn how to do software engineering was just _doing software engineering_. However, that's not an immutable fact of what software engineering is. It's a fortunate coincidence. Many jobs aren't like this.
